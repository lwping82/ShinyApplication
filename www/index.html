Introduction


This Shiny web application project was developed in order to complete the assignment required by Coursera's Course 9 - "Shiny Application and Reproducible Pitch".

The objective of this project is to accomplish the following:

  1. Making use of some form of input widget. Eg. textbox, radio button, checkbox, etc.
    • This requirement has been fulfilled through the use of buttons and selection input boxes available in both "Home" and "Statistics" tabs.
  2. Some operation on the ui input in sever.R.
    • Upon receiving a vote, server.R will process, format and then save the record to a text file.
  3. Some reactive output displayed as a result of server calculations.
    • This requiment has been fulfilled through the update of total vote received on the voted character as well as plotting and updating of statistics chart based on user's selections upon clicking on the "Submit" button.
  4. Include enough documentation so that a novice user could use your application.
    • This requiment has been fulfilled through "How it works?" section under "About" tab.


How it works?


Upon successfully loading of this Shiny web application, you will be brought to the default landing page, which is the "Home" tab.

Note : Since this is a mock up project, there is no restriction on how many votes your are allowed to cast on one or more characters at any one time.

To cast a vote to your favourite character:

  1. Click on "Home" tab
  2. Choose your favourite character, then click on "Vote" button
  3. Upon clicking on the button, a dialogue box will be prompted out. Click on "X" or "Close" button to close the dialogue box.
  4. Upon successfully processed the casted vote, the number of vote received on the character you just voted will be increased by 1

To view the statistics of overall votes:

  1. Click on "Statistics" tab
  2. Make your selections from the left pane then click on "Submit" button
  3. Upon receiving of your request, server.R will process the data then generate the chart accordingly based on the selected options
  4. In the event that there is no matching record found, a message will be displayed.
